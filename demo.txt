Example 1 — Classic factual override (entity replacement)
Question
Who is the CEO of Starbucks?
Context
Brian Niccol is the CEO of Starbucks.
Why this is a good demo
Strong parametric prior
Clean entity replacement
Easy to reason about
1️⃣ Static layer 14 (baseline)
python3 scripts/demo_budgeted_search.py \
  --model_id google/gemma-3-4b-it \
  --vectors_dir artifacts/gemma3_4b/vectors \
  --question "Who is the CEO of Starbucks?" \
  --context "Brian Niccol is the CEO of Starbucks." \
  --layer 14 \
  --multiplier 2.0 \
  --dynamic_layers false \
  --use_bplis false
Expected behavior
Output: Brian Niccol
Confident
No explanation
Works, but:
steering is blunt
same layer always
2️⃣ Old dynamic BCILS (no B-PLIS)
python3 scripts/demo_budgeted_search.py \
  --model_id google/gemma-3-4b-it \
  --vectors_dir artifacts/gemma3_4b/vectors \
  --question "Who is the CEO of Starbucks?" \
  --context "Brian Niccol is the CEO of Starbucks." \
  --dynamic_layers true \
  --use_bplis false
Expected behavior
Output: Brian Niccol
Chosen layer: 14
Identical to static
What you explain
“Although this is called ‘dynamic’, BCILS always picks layer 14 here. So behavior collapses to static.”
3️⃣ Full pipeline (BCILS + B-PLIS + Householder)
python3 scripts/demo_budgeted_search.py \
  --model_id google/gemma-3-4b-it \
  --vectors_dir artifacts/gemma3_4b/vectors \
  --question "Who is the CEO of Starbucks?" \
  --context "Brian Niccol is the CEO of Starbucks." \
  --dynamic_layers true \
  --use_bplis true \
  --bplis_generations 10 \
  --bplis_popsize 8 \
  --bplis_intrinsic_dim 64 \
  --householder_theta 0.6
Expected behavior
Output: Brian Niccol
used_search = true
Vector is query-specific
Possibly different layer (or same layer, but different vector)
What you explain
“Even when the layer stays the same, the vector is no longer static. This is true query adaptation.”

What you say
“We’re not correcting false context — we’re ensuring faithful adherence without collapse.”
How to explain the difference (one slide, one minute)
You can literally say this:
Static uses one vector at one layer for all queries.
Old dynamic (BCILS) selects a layer, but because representational dominance exists, it collapses to the same layer — behaving identically to static.
Our full pipeline keeps BCILS for where, but introduces B-PLIS for what, synthesizing a query-specific vector and injecting it via a norm-preserving rotation, which avoids overconfidence and improves robustness.
